## 问题：搜索功能关键字/户型/排序均不生效

### 现象
1. 前端搜索页（HouseList.vue）输入关键字搜索，返回结果未过滤，仍显示全部房源
2. 选择户型（如"2室"）后列表未筛选
3. 选择"价格从低到高"排序，列表仍按最新发布排列
4. URL 仅同步了 `keyword` 参数，其他筛选条件未体现在地址栏中

### 排查过程
1. 前端 `fetchHouses()` 确实将 `keyword`、`roomCount` 放入 axios params 发送
2. 后端 `HouseController.listHouses()` 接收了 `keyword`（通过 PageQuery）但**未传递给 service**
3. `HouseController.listHouses()` **完全没有** `roomCount` 和 `sortBy` 的 `@RequestParam`
4. `HouseServiceImpl.listPublishedHouses()` 方法签名**没有** `keyword`/`roomCount`/`sortBy` 参数
5. 实现中只有 city/district/price 筛选 + 固定 `orderByDesc(createTime)`，无模糊搜索、无户型筛选、无动态排序

### 根因
前后端搜索参数链路断裂：

| 参数 | 前端发送 | Controller 接收 | Service 处理 |
|------|---------|----------------|-------------|
| keyword | ✅ | ✅（PageQuery） | ❌ 未传递 |
| roomCount | ✅ | ❌ 无此参数 | ❌ 无此参数 |
| sortBy | ❌ 未发送 | ❌ 无此参数 | ❌ 无此参数 |
| city | ❌ 无下拉 | ✅ | ✅ |
| district | ❌ 无下拉 | ✅ | ✅ |

### 修复内容

**后端：**

1. `HouseService.java` — 接口签名扩展，`listPublishedHouses()` 新增 3 个参数：
   ```java
   Page<HouseInfo> listPublishedHouses(Page<HouseInfo> page,
       String city, String district,
       BigDecimal minPrice, BigDecimal maxPrice,
       String keyword, Integer roomCount, String sortBy);
   ```

2. `HouseServiceImpl.java` — 核心搜索逻辑实现：
   - **keyword 模糊搜索**：对 title、description、address、city、district 五个字段做 `LIKE '%kw%'`，用 `wrapper.and(w -> ...)` 包裹，内部 OR 连接
   - **roomCount 筛选**：`wrapper.ge(HouseInfo::getRooms, roomCount)`（"N室+"语义，选 2 室匹配 rooms >= 2）
   - **sortBy 动态排序**：`"price_asc"` → `orderByAsc(RentMonthly)`，`"price_desc"` → `orderByDesc(RentMonthly)`，其余 → `orderByDesc(CreateTime)`（最新发布）

3. `HouseController.java` — `listHouses()` 方法：
   - 新增 `@RequestParam(required = false) Integer roomCount`
   - 新增 `@RequestParam(required = false) String sortBy`
   - 将 `query.getKeyword()`、`roomCount`、`sortBy` 传递给 service

**前端：**

4. `api/house.ts` — `HouseQuery` 接口补充：
   ```typescript
   roomCount?: number
   sortBy?: string
   ```

5. `views/house/HouseList.vue` — 三项修复：
   - **sortBy 实际发送**：`fetchHouses()` 中非默认值时将 `sortBy` 添加到 API 请求参数
   - **city/district 发送**：`fetchHouses()` 中将 `city`/`district` 添加到 API 请求参数
   - **URL 全参数同步**：`handleSearch()` 同步所有有效筛选参数到 URL query（keyword/city/district/priceRange/roomCount/sortBy）；`query` 初始化时从 `route.query` 恢复所有筛选状态
   - **城市/区域下拉**：filter-items 中新增两个 `el-select`
     - 城市：广州 / 深圳 / 北京 / 上海
     - 区域：天河 / 海珠 / 番禺 / 白云 / 越秀 / 南山 / 福田
   - **resetFilters** 同步重置 city/district 字段

## 新增：简单支付功能（模拟支付 + 支付密码）

### 背景
PaymentOrder 表和实体已存在但完全独立于订单流程，前端无任何支付入口。订单在"房东确认"后即为死胡同，缺少"支付→完成"的业务闭环。采用支付密码验证方案，无需对接真实支付网关。

### 方案
1. 用户设置 6 位支付密码（BCrypt 加密）
2. 房东确认订单时自动生成支付单（押金 + 首月租金，24h 过期）
3. 租客在"我的订单"点击"去支付"→ 检测支付密码 → 输入密码验证 → 支付成功 → 订单完成

   - `PUT /user/payment-password` — 设置/修改支付密码（6位数字，首次设置无需旧密码）
4. `OrderController.java` — `confirmOrder()` 确认时自动创建 PaymentOrder，金额=押金+首月租金，过期时间 24h
5. `PaymentOrderController.java` — 重写 `pay` 接口：

   - 密码正确 → 标记 paid + paidAt，同步更新 RentOrder status→2（已完成）

**前端：**

6. `api/payment.ts`（新建）— `hasPaymentPassword()` / `setPaymentPassword()` / `getPaymentByRentOrder()` / `payOrder()`
7. `api/order.ts` — `RentOrder` 接口新增 `paymentOrderId`

## 新增：支付成功后发送订单信息邮件通知

### 背景
支付完成后租客无法获得支付凭证，需要将订单信息发送到用户邮箱作为记录。项目已引入 `spring-boot-starter-mail` 依赖，`OrderEmailService` 已有三个通知方法（新订单/确认/取消），SMTP 配置此前处于注释状态。

### 修复内容

**后端：**

1. `OrderEmailService.java` — 新增两个方法：
   - `sendPaymentSuccessNotification(UserInfo, RentOrder, PaymentOrder)` — 公开接口，校验 mailSender + 租客邮箱后发送 HTML 邮件
   - `buildPaymentSuccessEmail(...)` — 构建支付成功 HTML 邮件模板，包含支付单号、租房订单号、房源标题、费用类型、金额、支付时间、租期
   - 发件人地址改为 `@Value("${spring.mail.username}")` 注入，替代硬编码的 `noreply@rent-system.com`，避免 QQ 邮箱将邮件标记为垃圾邮件

2. `PaymentOrderController.java` — `pay()` 方法：
   - 注入 `OrderEmailService`（`@Autowired(required = false)`）
   - 支付成功后获取租客信息，调用 `sendPaymentSuccessNotification`，try-catch 包裹确保邮件失败不影响支付流程

3. `application.yml` — 激活邮件配置：
   - 删除底部重复的注释块
   - 将 `mail` 配置正确嵌套到已有 `spring:` 下（`servlet` 之后、`app` 之前）

**测试：**

4. `OrderEmailServiceTest.java`（新建）— 两个测试用例：
   - `testSendPaymentSuccessNotification` — 构造完整测试数据发送邮件
   - `testSendPaymentSuccessNotification_NoEmail` — 验证租客无邮箱时静默跳过

### 不处理的情况
- 租客未设置邮箱：静默跳过
- 邮箱不可用（无效地址/退信）：属异步退信，SMTP 同步发送时无法检测，需独立退信监控基础设施

## 搜索数据流（修复后）

```
用户操作                          前端                          后端
───────                          ────                          ────
输入"天河"点击搜索    →  handleSearch()                  →
                          更新URL: ?keyword=天河           →
                          调用 fetchHouses()              →
                          发送 GET /house/public/list      →
                          ?keyword=天河&page=1&size=12     →
                                                           →  Controller 接收参数
                                                              keyword=天河 (PageQuery)
                                                           →  Service.listPublishedHouses()
                                                              wrapper.and(w -> w
                                                                .like(title, "%天河%")
                                                                .or().like(description, "%天河%")
                                                                .or().like(address, "%天河%")
                                                                .or().like(city, "%天河%")
                                                                .or().like(district, "%天河%"))
                                                           →  返回匹配结果
                       ←  渲染列表                         ←

选择"2室" + "价格升序" → handleSearch()                  →
                          更新URL: ?roomCount=2           →
                          &sortBy=price_asc               →
                          调用 fetchHouses()              →
                          发送 GET ...&roomCount=2        →
                          &sortBy=price_asc               →
                                                           →  Controller 接收
                                                              roomCount=2, sortBy=price_asc
                                                           →  Service:
                                                              wrapper.ge(rooms, 2)
                                                              wrapper.orderByAsc(rentMonthly)
                                                           →  返回 rooms>=2 按价格升序的结果
                       ←  渲染列表                         ←
```
