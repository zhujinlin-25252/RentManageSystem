# Debug 记录 - 2026-06-03

## 问题 1：注册接口 400 Bad Request

### 现象
前端注册页面提交后，控制台报错：
```
api/auth/register:1  Failed to load resource: the server responded with a status of 400 (Bad Request)
VM50:2 1578845607
```

### 排查过程
1. 检查前端请求体字段 → 发送的是 `name`
2. 检查后端 DTO `RegisterRequest.java` → 字段是 `nickname`，且带有 `@NotBlank` 校验
3. Jackson 反序列化时 `nickname` 为 null → `@NotBlank` 校验失败 → 400

### 根因
前后端字段名不一致：
| 层级 | 字段名 |
|------|--------|
| 前端 form 模型 | `name` |
| 前端 store | `name` |
| 后端 RegisterRequest DTO | `nickname` |
| 后端数据库列 | `nickname` |

### 修复内容

**前端：**
- `RegisterPage.vue`：`name` → `nickname`（表单模型、校验规则、API 调用）
- `stores/user.ts`：`UserInfo.name` → `UserInfo.nickname`，`register()` 参数 `name` → `nickname`
- `MainLayout.vue`：`userInfo.name` → `userInfo.nickname`（头像和用户名显示）
- 密码校验增强：min:6 → min:8，增加字母+数字正则 `/^(?=.*[A-Za-z])(?=.*\d).+$/`

**后端：**
- `RegisterRequest.java`：新增 `role` 字段，使前端选择的租客/房东身份能正确传递
- `AuthController.java`：`role` 不再硬编码为 0，改为 `request.getRole() != null ? request.getRole() : 0`
- 新增 `GET /auth/check-phone?phone=xxx` 接口，供前端实时校验手机号是否已注册
- `SecurityConfig.java`：将 `/auth/check-phone` 加入公开白名单

### 涉及文件
- `rent-manage-frontend/src/views/auth/RegisterPage.vue`
- `rent-manage-frontend/src/stores/user.ts`
- `rent-manage-frontend/src/layouts/MainLayout.vue`
- `RentManageSystem/src/main/java/com/rent/managesystem/dto/RegisterRequest.java`
- `RentManageSystem/src/main/java/com/rent/managesystem/controller/AuthController.java`
- `RentManageSystem/src/main/java/com/rent/managesystem/config/SecurityConfig.java`

## 问题 2：前端手机号重复注册实时校验

### 需求
注册时输入手机号后，实时检查是否已被注册，并给出可视化反馈。

### 实现

**后端：** 新增 `GET /api/auth/check-phone?phone=xxx`
- 手机号格式非法直接返回 `false`
- 已注册返回 `true`，未注册返回 `false`

**前端：** `RegisterPage.vue`
- `@blur` + `@input` 触发 `checkPhone()`，500ms 防抖
- 三种状态反馈：检查中（灰色）、可用（绿色勾）、已注册（红色警告 + 跳转登录链接）
- 表单校验新增 `phoneExistsValidator`：手机号已注册或正在检查时阻止提交

## 问题 3：地址字段改为省市区级联选择

### 需求
房源发布页的省份/城市/区域原来为自由文本输入，无校验无规则，改为下拉级联选择 + 详细地址文本输入。

### 实现

**依赖：** 安装 `element-china-area-data` 包（提供完整中国省市区三级数据）

**前端：** `HousePublish.vue`
- 三个独立 `el-input`（province/city/district）→ 一个 `el-cascader`（region，值为 `['广东省', '广州市', '天河区']`）
- 表单模型：`{ province, city, district }` → `{ region: string[] }`
- 提交时 `region` 数组自动拆分为 `province`/`city`/`district` 传给后端
- 编辑回填时后端的三个字段合并为 `region` 数组
- 校验规则从三个必填改为一个必填 `{ required: true, message: '请选择所在地区', trigger: 'change' }`

**不需改动：**
- 展示类页面（HouseDetail/HouseList/HomePage/AuditPage/LandlordPage）仅展示已存储的字符串，数据格式未变
- 后端 HouseInfo 实体和 SQL 保持不变，仍然是三个 VARCHAR 字段
- 搜索筛选功能使用自由关键词搜索，与地址输入无关

### 涉及文件
- `rent-manage-frontend/src/views/landlord/HousePublish.vue`

## 问题 4：游客浏览模式

### 需求
未登录用户可以浏览房源，进行预订交互时弹窗提示登录/注册，登录后跳回原房源页面。

### 实现

**路由守卫：** `router/index.ts`
- `/home`、`/house`、`/house/:id` 添加 `meta: { public: true }`
- 游客访问 `/` 自动重定向到 `/home`，不被拦截
- 写操作路由（landlord/admin/tenant）保持需登录

**登录页：** `LoginPage.vue`
- 新增"以游客身份浏览房源"入口（虚线按钮，点击直接跳转 `/home`）

**房源详情页：** `HouseDetail.vue`
- "立即租房"按钮：登录用户正常打开申请对话框；游客弹出登录提示
- "收藏房源"按钮：同样增加游客判断
- 登录提示对话框：
  - "登录账号" → `/login?redirect=/house/{当前房源ID}`（登录后回到该房源）
  - "注册新账号" → 跳转注册页
  - "继续浏览，暂不登录" → 关闭对话框

**后端：** `SecurityConfig.java`
- `GET /api/upload/images/{houseId}` 加入公开白名单（修复游客浏览房源图片时 401 报错）

### 涉及文件
- `rent-manage-frontend/src/router/index.ts`
- `rent-manage-frontend/src/views/auth/LoginPage.vue`
- `rent-manage-frontend/src/views/house/HouseDetail.vue`
- `RentManageSystem/src/main/java/com/rent/managesystem/config/SecurityConfig.java`
