# Debug 记录 - 2026-06-11

## 问题：已出租房源从"我的订单"访问详情页报错

### 现象
当房源被出租（status=2，不再出现在首页）后，用户通过"我的订单"页面点击房源链接访问 `/house/:id`，页面显示"房源不存在或已下架"，无法查看房源详细信息。

### 排查过程
1. 跟踪"我的订单"页面 → 房源标题是 `<router-link to="/house/${order.houseId}">`
2. 详情页调用 `getHouseDetail(id)` → `GET /api/house/public/{houseId}`
3. 检查后端 `HouseController.getHouseDetail()` → 只允许 `status == 1`（已发布）的房源
4. 已出租房源 status=2，被拦截返回 `HOUSE_NOT_FOUND`

### 根因
后端公开详情接口只对已发布（status=1）房源开放，已出租（status=2）房源被拒绝访问。同时前端存在状态码映射错误，status 1/2/3 的中文标签与实际含义颠倒。

| status | 实际含义 | 前端错误映射 |
|--------|---------|-------------|
| 1 | 已发布 | 在租 |
| 2 | 已出租 | 已下架 |
| 3 | 已下架 | 已出租 |

### 修复内容

**后端：**

1. `ErrorCode.java` — 新增 `HOUSE_OFF_SHELF(310, "该房源已下架")` 错误码，用于区分"下架"和"不存在/已删除"

2. `HouseController.java` — 重写 `getHouseDetail()` 状态判断逻辑：
   ```
   null/deleted → HOUSE_NOT_FOUND (301)
   status=3     → HOUSE_OFF_SHELF (310)
   status=1,2   → 返回详情（已发布 + 已出租均可查看）
   status=0,4   → HOUSE_NOT_FOUND (301)
   ```

**前端：**

3. `request.ts` — 响应拦截器增强：
   - 支持请求配置 `_silent: true`，调用方可跳过自动 error toast
   - 错误对象附带 `code` 属性，组件可据此区分错误类型

4. `api/house.ts` — `getHouseDetail(id, config?)` 新增可选 config 参数，支持传入 `{ _silent: true }`

5. `views/house/HouseDetail.vue` — 核心改造：
   - 引入 `pageState` 状态机：`'loading'` | `'active'` | `'off_shelf'` | `'not_found'`
   - `pageState === 'active'` → 正常展示房源详情
   - `pageState === 'off_shelf'` → 展示"该房源已下架"（错误码 310）
   - `pageState === 'not_found'` → 展示"房源不存在或已被删除"
   - 已出租（status=2）房源："立即租房"按钮替换为禁用的"该房源已被出租"
   - 修复 `getStatusText()` 状态映射：`0→待审核, 1→已发布, 2→已出租, 3→已下架, 4→审核拒绝`
   - 新增 `statusTagType()` 按状态返回正确的 el-tag 颜色

6. `views/home/HomePage.vue` — 修复 `getStatusText()` 状态映射（同上）

7. `views/house/HouseList.vue` — 修复 `getStatusText()` 状态映射（同上），同步更新 CSS 状态标签颜色：
   - status-0: #f39c12（待审核-橙色）
   - status-1: 绿色渐变（已发布）
   - status-2: #3498db（已出租-蓝色，原为红色 #e74c3c）
   - status-3: #95a5a6（已下架-灰色）
   - status-4: #e74c3c（审核拒绝-红色，新增）

## 房源详情页状态流转图

```
用户访问 /house/:id
  │
  ├─ houseId 无效 ──→ pageState = 'not_found'  → "房源不存在或已被删除"
  │
  ├─ API 返回成功 ──→ pageState = 'active'      → 展示房源详情
  │   ├─ status=1 (已发布) → 正常展示 + "立即租房"按钮可用
  │   └─ status=2 (已出租) → 正常展示 + "该房源已被出租"按钮禁用
  │
  ├─ API 返回 310  ──→ pageState = 'off_shelf'  → "该房源已下架"
  │
  └─ API 返回其他错误 → pageState = 'not_found' → "房源不存在或已被删除"
```

## 问题：房东查看自己房源详情时显示租客操作按钮 + 非公开状态房源无法查看

### 现象
1. 房东从房源管理页点击"查看"进入 `/house/:id`，右侧仍显示"立即租房"和"收藏房源"按钮，房东不需要这些功能
2. 审核未通过（status=4）或待审核（status=0）的房源，公开接口拒绝访问，显示"房源不存在"，房东无法预览自己未发布的房源

### 根因
- `HouseDetail.vue` 始终使用公开接口 `GET /api/house/public/{houseId}`，该接口只允许 status=1/2
- 页面未区分"房东管理视角"和"租客浏览视角"，所有用户看到相同的操作面板

### 修复内容

**1. API 层增强**
- `HouseInfo` 接口新增 `landlordId`、`auditRemark`、`auditTime` 字段
- `getMyHouseDetail(id, config?)` 支持传入 axios config（用于 silent 模式）

**2. fetchDetail 双重获取策略**
```
公开接口成功 → 租客视角，正常展示
公开接口失败 + 用户已登录 → 尝试房东接口 getMyHouseDetail
  ├─ 房东接口成功 → 房东视角（isLandlordView = true）
  └─ 房东接口失败 → 使用公开接口的错误码展示错误页
```

**3. 房东管理面板（isLandlordView = true 时显示）**
- 状态描述文案：根据 status 0-4 显示不同的管理提示
- 操作按钮：
  - 编辑房源 / 修改后重新提交（status=4 时文案自动切换）
  - 下架（status=1/2 时显示）
  - 重新上架（status=3 时显示）
  - 返回房源管理
- 审核拒绝时显示拒绝原因（红色提示框）
- 租客视角的"立即租房""收藏房源"按钮完全隐藏

**4. 租客视角保持不变**
- 非房东用户看到的界面与之前完全一致
- 房东信息卡片仅在租客视角显示

### 涉及文件
- `rent-manage-frontend/src/api/house.ts`
- `rent-manage-frontend/src/views/house/HouseDetail.vue`

### 追加修复（同日）
